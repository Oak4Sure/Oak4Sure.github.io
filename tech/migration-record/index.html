How to manage one more github accounts in your computer? 解决一台电脑多Github账号共存问题。

Today I tried to migrate my blog from the original GitHub repository to a new one to separate my work account from my personal account. However, it took me quite a while.

#Create a new Github account and set up the SSH key

Don't use the same key as the old GitHub account. It will report that the public key is already in use. So, you had to generate a new SSH key for the new account. ssh-keygen -t rsa -C "Your new email" -f "new key's path with a new name" After that, use the new public key to set up your GitHub.

#Add the private key to ssh

1
2

ssh-add ~/.ssh/id_rsa_one
ssh-add ~/.ssh/id_rsa_two

#Modify _config.yml

In the deploy section, there is a repo key. Change the repo key to the new address.

1
2
3

hexo clean
hexo g
hexo d

However, it reported that ERROR: Permission to Oak4Sure/Oak4Sure.github.io.git denied to old account.
fatal: Could not read from remote repository. Please make sure you have the correct access rights and the repository exists.

That's because your SSH hasn't been updated with the new public key; it is still using the old public key for the old account.

#Modify ssh config

I searched for a solution, and they suggested modifying the ~/.ssh/config file. However, it didn't work. Because as u modify the config file, and U named ur both github accounts, u should specify the nickname on the _config.yml

#Example

When u copy a ssh link on the github, u will get the formation as: git@github.com:UserName/RepositoryName.git U should modify it as: git@HostName.github.com:UserName/RepositoryName.git so that when u try to make a ssh connection, the ssh will parse the link using the config file u've already modified.
