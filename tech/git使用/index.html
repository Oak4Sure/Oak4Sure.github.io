进了企业不会Git的都是假程序员，为我目前接触到的git常用命令做一个总结。

Git的三个区：

  • 工作区，你看到的目录和代码
  • 暂存区
  • Commit区/本地仓库

#工作区到暂存区

1
2

git add .
git add <filename>

#暂存区到本地仓库

1

git commit -m <string>

#更改最后commit的说明

1

git commit --amend

#查看工作区和暂存区的区别

1

git diff

#查看暂存区和本地仓库的区别

1
2

git diff --staged
# 还有个`--cached`参数和`--staged`作用一样

#Git状态查看

1

git status

这算是最常用的命令，它会告诉你：

  • 哪些文件未staged
  • 哪些文件未commit
  • 当前开发所在分支

#Git分支

#切换本地分支

1
2
3
4
5
6
7

git checkout <branch>
git checkout local_ID
# 新建并切换到分支
git checkout -b <new_branch>
# 新建本地分支并跟踪对应远端分支
git checkout -b <local_branch> <remote_name>/<remote_branch>
git checkout -b Br1_local origin/Br_1

#做本地分支代码修改并commit到本地仓库

1
2

git add .
git commit -m <string>

#查看远端仓库和远端分支名字

1
2
3
4
5
6

# 查看远端仓库名字
git remote
# 详细版本
git remote -v
# 查看本地仓库和远端仓库的进度
git log

#推送远端

1
2
3
4

git push <remote> <remote_branch>
# 建立起本地分支和远端分支的对应
git push <remote> <local_branch>:<remote_branch>
git push origin local_ID:br_ID

#回退更改(未staged的内容)

1
2
3
4

# 回退指定文件
git checkout -- <file>
# 回退当前目录下所有文件
git checkout -- .

#查看commit内容

1
2
3
4
5
6
7
8

# 默认最新一次commit
git show
# 指定commit
git show <commit-hash>
# 查看某次commit文件的内容
git show <commit-hash>:<file-path>
# 查看某次commit特定文件的差异
git show <commit-hash> -- <file-path>
